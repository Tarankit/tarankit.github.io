CASE 01 — DIGITOLL

Integrating a nation's customs API without hardcoding a single identity

Norway's Digitoll requires per-broker machine identities with certificate-bound JWT grants. Early incoherence between certs and scopes caused failed token mints that looked like outages.

ProblemMulti-broker OAuth identity across road, rail & air transport scopes
ArchitectureCoherent identity model — cert, issuer and scope resolved together from Azure Key Vault per broker
ImpactRail clearance unlocked for a national transport operator, zero credential incidents
POST /token · maskinporten.no
grant: urn:ietf:params:oauth:jwt-bearer
cert: keyvault://9-Cert-MaskinPorten ✓
scope: toll:movement/road toll:movement/rail
→ 200 OK · access_token granted
clearance.status: ACCEPTED · 214ms
token mint — production trace
CASE 02 — GDS AI

Shipping an LLM assistant that's grounded, not a gimmick

Customs data is unforgiving — hallucinated tariff codes cost money. The assistant runs a tool-use loop: the model calls typed tools, results are grounded in platform data, and every answer degrades gracefully.

ProblemSupport load on tariff classification and "why is my shipment red?"
ArchitectureFunction-calling loop in C# with grounded tariff tools + status-aware onboarding automation
ImpactManual onboarding support cut dramatically; users self-serve answers
user query→ LLM→ classify_tariff()→ LLM→ grounded answer
tool-use loop — no ungrounded claims
CASE 03 — IMPORT PIPELINE

A pipeline that learns from its mistakes — carefully

OCR'd invoices feed real declarations. The system records human corrections and reapplies them — but a naive version once learned a wrong country code from a legend digit. I built the guards that make self-learning safe.

ProblemBad references and OCR noise reaching customs declarations
ArchitectureReference sanitization, dumped-row protection, letterless-source guards on learned corrections
ImpactReduced manual corrections → improved declaration quality → regression prevented — corrections compound instead of decaying
import invoice_88214.pdf
ref "3" → country? ✗ blocked (letterless source)
ref "SE556677" → ✓ sanitized · kept
learned: multi-Tull-ID lists → ✓ preserved
→ declaration clean · 0 guessed codes
correction guards — production behavior
